// History: VULKAN_RenderFrame.cpp — 30-05-2026
// Changes made to RenderGamePlay():
//
// CRITICAL BUG FIX — Model render array was wrong:
//   Was:  for (int i = 0; i < MAX_MODELS; ++i) { Model& m = models[i]; ... }
//   Now:  for (int i = 0; i < MAX_SCENE_MODELS; ++i) { Model& m = scene.scene_models[i]; ... }
//
//   The Vulkan GPU buffers (vertexBuffer, indexBuffer, uniformBuffer, descriptorSet)
//   are set up by SetupModelForRendering() on scene.scene_models[] entries (the
//   scene-instanced array). The global models[] cache entries have VK_NULL_HANDLE
//   for all GPU handles. Iterating models[] meant NOTHING was ever drawn.
//
// GUARD FIX — Transform-proxy / transform-only node skip:
//   Was:  if (!m.IsActive() || !m.HasGeometry()) continue;
//   Now:  if (!m.m_isLoaded) continue;
//         if (m.m_modelInfo.bIsTransformProxy || m.m_modelInfo.bIsTransformOnly) continue;
//   Matches DX11 RenderGamePlay exactly; these nodes are GLTF hierarchy empties with no geometry.
//
// DX11 PARITY — fxActive reset:
//   Added: m.m_modelInfo.fxActive = false; per model, matching DX11 RenderGamePlay.
//
// PER-MODEL TEXTURE BINDING (new feature — parity with DX11 texture binding):
//   Was:  always bind m_defaultTextureDescSet (1x1 white for every model)
//   Now:  if model has a loaded texture with a valid VkImageView, allocate a
//         temporary descriptor set using that view + m_defaultTexture.sampler,
//         bind it at set=0, and defer-free via fd.pendingFreeSets.
//         Falls back to m_defaultTextureDescSet when no texture is available.
//
// OPTIMIZATION — Light push constant moved outside draw loop:
//   Was:  lightsManager.GetAllLights() called inside the per-model loop (N calls/frame)
//   Now:  called once before the loop; single vkCmdPushConstants() before the draw loop.
//         The push constant is per-pipeline-bind, not per-draw-call; pushing it once is correct.
//
// CLEANUP — Removed redundant platform-duplicate light blocks:
//   The PLATFORM_WINDOWS and PLATFORM_LINUX/ANDROID blocks were byte-for-byte identical.
//   Replaced with a single cross-platform block (sun.direction, sun.color, etc. are
//   cross-platform via XMFLOAT3-aliased Vector3 in LightStruct).
