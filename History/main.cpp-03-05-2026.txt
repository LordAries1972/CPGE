--- 03-05-2026 13:30 AEST ---
Display mode startup now driven by config.myConfig.displayMode (0=Windowed, 1=Borderless, 2=Fullscreen).

Replaced the hard-coded #if defined(_DEBUG) / #else / #endif block (old lines ~539-557) that always went
windowed in Debug and fullscreen in Release.  The new switch block applies the correct window state and
cursor policy for all three modes immediately after config.loadConfig():

  case 0 (Windowed):
    - winMetrics.isFullScreen = false, winMetrics.isBorderless = false
    - Calls AdjustWindowRect + SetWindowPos to resize client area to config resolution, then re-centres.
    - No ShowCursor call; WM_SETCURSOR handler (see below) hides cursor in client area only.

  case 1 (Borderless):
    - winMetrics.isFullScreen = false, winMetrics.isBorderless = true
    - Calls SetWindowLong(GWL_STYLE, WS_POPUP|WS_VISIBLE) to strip title bar / borders.
    - Calls SetWindowPos(HWND_TOP, 0,0, SM_CXSCREEN, SM_CYSCREEN) to fill primary monitor.
    - WM_SETCURSOR hides cursor inside the window; Windows shows it automatically outside.

  default / case 2 (Full Screen exclusive):
    - winMetrics.isFullScreen = true, winMetrics.isBorderless = false
    - Calls sysUtils.DisableMouseCursor() (ShowCursor FALSE) to fully suppress Windows cursor.
    - Calls renderer->SetFullExclusive(resolutionWidth, resolutionHeight).

StartRendererThreads() is now called immediately after the display mode switch closes (before soundManager,
moviePlayer, joystick, callback setup).  The loader thread sets bLoaderTaskFinished=false on startup, so
the loader ring is visible during the entire remaining init sequence — same behaviour as the old early
Debug call at line ~542.  The late unconditional call that was placed after the config callbacks was
removed; there is now exactly one call site.

Added WM_SETCURSOR case to WindowProc (before WM_MOUSEMOVE):
  - When !winMetrics.isFullScreen and hit-test == HTCLIENT: calls SetCursor(nullptr), returns TRUE.
  - This suppresses the Windows arrow cursor inside the client area for Windowed and Borderless modes
    while leaving non-client areas (title bar, resize borders) untouched for Windowed mode.
  - For Fullscreen the message is moot (ShowCursor already suppressed the cursor system-wide).

--- 03-05-2026 (follow-up) ---
Fix: loader ring invisible in Windowed and Borderless modes.

Root cause: SetWindowPos calls in cases 0 and 1 of the display mode startup switch were firing
WM_SIZE synchronously before StartRendererThreads().  WM_SIZE triggered Resize() which called
Clean2DTextures() and reset m_d2dRenderTarget, wiping all D2D state before the loader thread
had loaded BG_LOADER_CIRCLE (loadingring.png).  Ring drawing code checks both
!bLoaderTaskFinished AND m_d2dTextures[BG_LOADER_CIRCLE] non-null; the latter was null so the
ring was never drawn.

Fix applied:
  case 0 (Windowed): removed AdjustWindowRect + SetWindowPos + CenterSystemWindow from the
    switch body; only winMetrics flag assignments remain.

  case 1 (Borderless): removed SetWindowPos from the switch body; SetWindowLong style change
    remains because it does not alter window size and does not fire WM_SIZE.

  After StartRendererThreads() returns, window geometry is applied:
    - Borderless: SetWindowPos(HWND_TOP, 0, 0, monW, monH, SWP_FRAMECHANGED)
    - Windowed:   AdjustWindowRect + SetWindowPos(SWP_NOMOVE|SWP_NOZORDER|SWP_FRAMECHANGED)
                  + CenterSystemWindow

  WM_SIZE now fires with threads already running, so Resize() executes through the designed
  concurrent path (loader mutex protects texture state during reload).

Also fixed C4715 warning in WM_SETCURSOR handler:
  The non-HTCLIENT path previously fell through via break to the end of WindowProc with no
  return value.  Changed to: return DefWindowProc(hwnd, uMsg, wParam, lParam), which is also
  semantically correct — lets Windows handle cursor for non-client regions.
