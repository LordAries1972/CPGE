Recording start: replaced GetClientRect with swap chain GetDesc1() for dimensions.

GetClientRect and DXGI can return different values on DPI-aware systems.
CreateSwapChainForHwnd(Width=0, Height=0) lets DXGI size the back buffer from
the HWND's logical DPI coordinates. On a per-monitor-DPI-aware app, GetClientRect
returns physical pixels, which can be larger than what DXGI chose for the back
buffer. StartRecording was storing these larger values as m_width/m_height, and
every CaptureFrame call immediately hit the "back buffer smaller than recording
dimensions" guard and skipped every frame — resulting in no video in the recording.

Fix: query IDXGISwapChain1::GetDesc1() at recording start to get the actual
back buffer dimensions (scDesc.Width × scDesc.Height). These are guaranteed to
match bbDesc.Width × bbDesc.Height in CaptureFrame. Falls back to GetClientRect
if GetDesc1 fails.
