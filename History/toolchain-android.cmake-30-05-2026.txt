NEW FILE — Linux/toolchain-android.cmake
Thin wrapper around the Android NDK's official android.toolchain.cmake.

Purpose:
- Centralises Android build defaults (ABI, API level, STL) in one file
- Auto-discovers the NDK from ANDROID_NDK, ANDROID_NDK_HOME, or ANDROID_HOME env vars
- Provides a clear FATAL_ERROR with setup instructions when the NDK is missing
- Delegates to $ANDROID_NDK/build/cmake/android.toolchain.cmake which is
  the canonical NDK-provided toolchain; this wrapper adds nothing to it except
  auto-discovery and default variable setup

Defaults:
  ANDROID_ABI      = arm64-v8a         (modern 64-bit ARM devices)
  ANDROID_PLATFORM = android-24        (Android 7.0, minimum for Vulkan 1.0)
  ANDROID_STL      = c++_shared        (full C++17 standard library)

Usage (alternative to cmake-build-android.sh):
  cmake -DCMAKE_TOOLCHAIN_FILE=Linux/toolchain-android.cmake \
        -DANDROID_NDK=/path/to/ndk/<version>                  \
        -DANDROID_ABI=arm64-v8a                               \
        -DRENDERER=Vulkan                                      \
        -S Linux -B Linux/build-android/arm64-v8a/Vulkan/Debug
  cmake --build Linux/build-android/arm64-v8a/Vulkan/Debug --parallel
