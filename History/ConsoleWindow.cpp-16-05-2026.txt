ConsoleWindow.cpp - Created / Updated 16-05-2026
=================================================
Initial creation (see earlier entry in this file).

Bug-fix update (same day):
  - COLOURS: All fill colours were near-black (e.g. titlebar RGB 28,28,36 and content RGB 10,10,14),
    making the window invisible against the game's dark starfield/space background.
    Fixed by using a blue-themed palette:
      Outer border  : MyColor(60, 140, 220, 230)  — bright blue, visible on any background
      Titlebar bg   : MyColor(20, 55, 100, 245)   — dark blue
      Titlebar text : MyColor(220, 235, 255, 255) — bright near-white
      Titlebar sep  : MyColor(60, 120, 190, 255)  — medium blue
      Content bg    : MyColor(12, 18, 30, 218)    — dark blue-black semi-transparent
      Scrollbar trk : MyColor(18, 35, 60, 255)    — dark blue
      Scrollbar sep : MyColor(55, 110, 175, 255)  — medium blue
      Scrollbar thumb: MyColor(80, 150, 225, 255) — bright blue thumb

Position fix (same day):
  - Bottom margin increased from 10.0f to 50.0f so the console clears the Windows taskbar
    and the full window bottom is visible.

Debug-console integration update (same day):
  - AddLine() accepts ConsoleLineColor type; stores ConsoleLine{text, type} in m_buffer
  - Render() resolves per-line color at draw time:
      Normal  → MyColor(210, 210, 210, 255)  white
      Warning → MyColor(255, 220,   0, 255)  yellow
      Error   → MyColor(255, 140,   0, 255)  orange

Scrollbar fix + resize update (same day):
  - Added Scroll(int delta): clamps m_scrollOffset by ±delta within [0, maxOffset] under mutex
  - Window width clamp raised from (0.45, 260–450) to (0.52, 320–600) — wider window
  - Height grows automatically from CONSOLE_VISIBLE_LINES 10→14 change in header
  - PAGE_UP / PAGE_DOWN scrolling wired in KBHandlersCode.cpp

Width + no-wrap update (same day):
  - Window width clamp raised from (0.52, 600–600) to (0.80, 600–1400) — much wider
  - Text draw call switched from DrawMyText(text, pos, size, color, fontSize) to
    DrawMyText(text, pos, color, fontSize) — no bounding box so text never wraps

Mouse input update (same day):
  - Render() caches m_sbX, m_sbY, m_sbH after computing scrollbar geometry each frame
  - HandleMouseWheel(int delta): when visible, calls Scroll(+3) or Scroll(-3)
  - HandleMouseClick(float x, float y): hit-tests against cached scrollbar rect;
    maps click Y position to scroll offset (top=oldest, bottom=newest)
