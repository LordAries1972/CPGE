WarpDotTunnel enhancements — 16/05/2026

Init3DWarpDOTTunnel:
- Each ring now assigned ring.colorStep = i % kGraySteps so the tunnel starts with a staggered sequential gray pattern from first frame.

UpdateWarpDotTunnel:
- sideWaveTime advanced by dt each frame.
- Furthest ring (max zPos) identified by index scan each update.
- Furthest ring's cx = startX + kSideWaveRadius * sin(sideWaveTime * kSideWaveSpeed), giving ±30-unit left/right oscillation.
- All other rings keep cx = startX.
- Speed curve forward changed from quadratic (0.1 + t^2 * 2.4) to cubic (0.05 + t^3 * 3.9) — more dramatic near/far ratio.
- Speed curve reverse changed from linear (0.2 + t * 2.3) to quadratic (0.1 + t^2 * 3.5) — faster near, slower away.
- Loop changed from range-for to index-for to allow furthest-ring identification by index.

RenderWarpDotTunnel:
- Replaced blue-tinted brightness calculation with 8-step gray ramp lookup keyed on ring.colorStep.
- Gray ramp: {0.08, 0.19, 0.30, 0.44, 0.58, 0.72, 0.86, 1.0} — very dark grey to white.
- r = g = b = gray; alpha and dotSize logic unchanged.
