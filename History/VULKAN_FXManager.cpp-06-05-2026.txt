VULKAN_FXManager.cpp - 06-05-2026
===================================
NEW FILE

VKFXManager full implementation — Vulkan port of DX_FXManager.cpp.

Key additions:
- CreateFadePipeline(): Vulkan graphics pipeline for fullscreen fade quad using
  inline GLSL (fullscreen triangle from gl_VertexIndex, push constant vec4 color),
  compiled at runtime via shaderc; guarded by HAS_SHADERC macro
- DestroyFadePipeline(): vkDestroyPipeline / vkDestroyPipelineLayout
- RenderFadeFullScreenQuad(cmd, color): bind fade pipeline, push 16-byte color constant,
  vkCmdDraw(cmd, 3, 1, 0, 0)
- ApplyColorFader(): same progress/alpha math as DX11; calls GetCurrentCommandBuffer()
  from VulkanRenderer to get active cmd buffer
- Render() / Render2D(): same logic as DX11 with m_ prefixed member names
- RenderFX(): per-effect dispatch for ColorFader and Starfield with VkCommandBuffer
- Full starfield: CreateStarfield / UpdateStarfield / StopStarfield / RenderStarfield
  — uses VKParticle.particles (not VKStar.stars), same trajectory math as DX11;
  Linux/Android uses GLM-based projection instead of DirectXMath
- Particle explosion: CreateParticleExplosion / RenderParticles — identical to DX11
- All scroll helpers: StartScrollEffect, StopScrollEffect, UpdateScrollSpeed, PauseScroll,
  ResumeScroll, SetScrollDirection, FadeScrollSpeed, UpdateTweens, StartParallaxLayer,
  ApplyScroller
- Text scrollers: CreateTextScrollerLTOR/RTOL/Consistent/Movie; StopTextScroller,
  PauseTextScroller, ResumeTextScroller; UpdateTextScroller; RenderTextScroller
- CalculateTextTransparency / CalculateCharacterTransparency / SplitTextIntoLines
- CalculateTextWidthWithSpacing
- StopAllFXForResize / RestartFXAfterResize with SecureZeroMemory on Windows,
  aggregate init on Linux/Android
- Callback processing: same two-pass execute/remove pattern as DX11
- RemoveCompletedEffects: same two-pass index approach as DX11
