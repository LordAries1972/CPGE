DX11Renderer.cpp - 03-05-2026
==============================
Fix 1: Loader ring not visible during loading — pre-load bootstrap textures.
In StartRendererThreads(), before starting the loader thread, LoadTexture() is
called for BLIT_ALWAYS_CURSOR (index 0) and BG_LOADER_CIRCLE (index 2).
m_d2dRenderTarget is confirmed valid at that point (bIsInitialized is true).

Fix 2: Loader ring not visible — D2D thread safety race in LoadTexture().
Root cause: the loader thread called m_d2dRenderTarget->CreateBitmapFromWicBitmap()
and assigned m_d2dTextures[idx] = bitmap with NO lock, while the render thread
concurrently called DrawBitmap(m_d2dTextures[idx].Get()) inside d2dRenderLock.
D2D is not thread-safe. This concurrent access caused D2D to enter an error state;
EndDraw() returned failure and the ring (and all 2D) became invisible — silently
in release builds.
Fix: wrapped CreateBitmapFromWicBitmap + m_d2dTextures assignment in a
ThreadLockHelper(D2DLockName, 5000). WIC file I/O (slow) happens outside the
lock; only the brief D2D GPU call and ComPtr assignment are serialized.

Fix 3: Aspect ratio not updated on resize.
Camera::UpdateResolution() existed but was never called from DX11Renderer::Resize().
Fix: after iOrigWidth/iOrigHeight are updated, calls:
    LookupAspectRatio(width, height) → config.myConfig.aspectRatio → myCamera.UpdateResolution()

Fix 4: DXRenderFrame.cpp — removed dead BeginDraw try-catch.
ID2D1RenderTarget::BeginDraw() returns void; it cannot throw std::exception.
The catch block was dead code. D2D errors are reported at EndDraw() HRESULT.
BeginDraw() is now called directly with an explanatory comment.
