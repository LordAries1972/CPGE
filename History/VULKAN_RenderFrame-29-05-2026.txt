   block called myCamera.GetPosition()/GetViewMatrix() expecting XMFLOAT3/XMMATRIX
   but these always return glm::vec3/glm::mat4.
   - Unified to: glm::vec3 camPos / glm::mat4 view / glm::mat4 proj (all platforms)
   - Y-flip: proj[1][1] *= -1.0f (was XMVectorNegate on proj.r[1])
   - UBO upload: world matrix uses XMStoreFloat4x4+memcpy (Windows DirectXMath);
     view/proj use memcpy from glm::value_ptr(glm::transpose(...))
   - FPS coords: unified to glm::vec3 (was platform-split struct)
   - RenderFX call: GetViewMatrix() (correct method name)
   - glm::perspective<float>() to fix template deduction ambiguity
