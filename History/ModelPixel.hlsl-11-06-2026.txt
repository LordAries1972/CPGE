ModelPixel.hlsl - History - 11 June 2026
=========================================

Changes made on 11-06-2026:

FIX - FXC X4000 "use of potentially uninitialized variable" warnings:

  Two warnings were emitted by the FXC shader compiler (same HRESULT 0x80070057 session):

  1. ProcessLight (line 203): warning X4000 at the early return "if (light.active == 0) return..."
     FXC's dataflow analysis does not track that outDiff and outSpec are assigned
     unconditionally at the top of the function before any branch.  It warns at each
     early-return statement that the out parameters might be unset.

     Fix: removed both early returns (active==0 and NdotL<=0.0001).  Replaced with
     nested [branch] if-blocks so the function has a single return at the bottom:
       return outDiff + outSpec;
     FXC can now clearly prove outDiff and outSpec are always valid at the return.

  2. SampleShadow (line 271): warning X4000 at the early return "return 1.0f"
     FXC flagged this at the first control-flow exit in the function.  Root cause:
     shadowUV was declared as "float2 shadowUV;" and then assigned component-by-component:
       shadowUV.x = ...; shadowUV.y = ...;
     FXC treats component-wise assignment differently from full initialisation and
     reports the variable as potentially uninitialised.

     Fix: changed to constructor initialisation:
       float2 shadowUV = float2(projCoords.x * 0.5f + 0.5f, projCoords.y * -0.5f + 0.5f);

  Logic and output are unchanged; only control-flow structure differs.
