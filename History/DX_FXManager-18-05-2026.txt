DX_FXManager.cpp - History - 18-05-2026
========================================

BUG FIX: #include "Includes.h" moved before outer preprocessor guard

Root cause of ALL FXManager unresolved-external-symbol linker errors:

  Line 1 of the file was:
      #if defined(__USE_DIRECTX_11__)

  And #include "Includes.h" was on line 4, INSIDE that guard.

  __USE_DIRECTX_11__ is defined inside Includes.h — not globally.
  At the point where line 1 was evaluated, no headers had been
  processed yet, so __USE_DIRECTX_11__ was undefined, the guard
  evaluated to FALSE, and the ENTIRE file was skipped.
  Result: empty .obj, every FXManager method became an
  unresolved external symbol.

  OpenGLFXManager.cpp and VULKAN_FXManager.cpp already had the
  correct pattern (Includes.h first, guard second).

Fix applied:
  #include "Includes.h" moved to the very top of the file,
  before the #if defined(__USE_DIRECTX_11__) guard.
  The guard and all DX11-specific code remain as-is.

Files changed: DX_FXManager.cpp
