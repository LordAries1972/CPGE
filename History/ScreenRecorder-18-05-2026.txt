ScreenRecorder.cpp - History - 18-05-2026
==========================================

CHANGE: InitMicCapture() — Prefer eCommunications capture endpoint over eConsole

Previously InitMicCapture() called GetDefaultAudioEndpoint(eCapture, eConsole, ...) which
selects the Windows "Default Device" microphone. When a headset is plugged in, Windows
typically assigns the headset microphone to the "Default Communication Device" role
(eCommunications) while the built-in laptop/desktop microphone remains as the "Default Device"
(eConsole). This caused the wrong microphone (built-in) to be captured instead of the headset.

Fix: Now tries eCommunications first, falls back to eConsole if no communications endpoint
exists. This mirrors the existing InitMonitor() logic which already correctly preferred
eCommunications for the render (playback) endpoint. A debug log message now reports which
endpoint was selected (Communications or Console) so it is visible in the log.
