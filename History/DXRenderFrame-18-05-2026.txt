FIX (v3): Renderer info overlay — exact positioning via IDWriteTextLayout::GetMetrics()

Root cause of text still not appearing after v2:
  DrawMyText() internally creates a layout rect of position.x + 1000.0f as the right
  boundary. When position.x is near the right edge of the canvas, this layout rect
  extends far beyond the D2D render target, and DirectWrite silently clips the draw
  call — rendering nothing visible.

Fix applied:
  Bypassed DrawMyText() entirely. Now uses the DirectWrite pipeline directly:
    1. GetOrCreateTextFormat(FontName, riFontSize)    — cached format
    2. m_dwriteFactory->CreateTextLayout(...)         — layout sized to full iOrigWidth
    3. riLayout->GetMetrics(&riMetrics)               — ACTUAL width and height of text
    4. riX = iOrigWidth  - riMetrics.width            — text starts textWidth from right edge
    5. riY = iOrigHeight - riMetrics.height           — text starts textHeight from bottom
    6. m_d2dRenderTarget->DrawTextLayout(...)         — direct D2D draw, no clipping

  Colour set via m_pixelBrush with D2D1::ColorF(220/255, 220/255, 220/255, 1.0f).

  X and Y are exact: no margin, no padding, no estimate. The right edge of the rendered
  text aligns to iOrigWidth; the bottom edge aligns to iOrigHeight.

Files changed: DXRenderFrame.cpp

FIX (v2): Renderer info overlay — coordinate system and visibility corrections

Root causes of text not appearing:
  1. Used `height` (from GetClientRect) instead of `iOrigHeight` (D2D render target
     coordinate space). All other bottom-edge elements (loading indicator) correctly
     use iOrigWidth/iOrigHeight. `GetClientRect` height excludes the window title bar,
     so the Y coordinate was off from the actual D2D canvas bottom.
  2. Font size was too small (8pt minimum) and colour too dim (160,160,160 at 200/255
     alpha) to be reliably visible on most game backgrounds.

Fixes applied:
  - X: iOrigWidth  - textWidth - 10px  (right edge, text anchored left of it)
  - Y: iOrigHeight - fontSize  - 10px  (bottom edge of D2D canvas)
  - Font: clamp(iOrigHeight / 72.0f, 10.0f, 16.0f)  — 10pt at <=720p, ~15pt at 1080p
  - Colour: MyColor(220, 220, 220, 255)  — bright white, fully opaque

ORIGINAL NEW: #include "BuildInfo.h" added to DX11 include block

Required so CURRENT_BUILD_VERSION, CURRENT_BUILD_SUBVERSION, and
CURRENT_BUILD constants are available within RenderFrame() for the
renderer info overlay.

CHANGED: Debug text font size now resolution-scaled

The FPS/debug info DrawMyText call used a hardcoded font size of 10.0f
regardless of screen resolution. Updated to use the same formula as the
console window:

    const float dbgFontSize = std::clamp(height / 108.0f, 8.0f, 12.0f);

Results by resolution:
    600p  ->  ~5.6  -> clamped to  8.0 pt
    720p  ->  ~6.7  -> clamped to  8.0 pt
    1080p ->  10.0  ->            10.0 pt
    1440p ->  13.3  -> clamped to 12.0 pt
    2160p ->  20.0  -> clamped to 12.0 pt

NEW: Renderer info overlay — bottom-right corner

Added immediately after the FPS debug block, toggled by USE_RENDERER_INFO
in Renderer.h (default: true).

Displays: "DirectX 11 v0.0.1236" in the bottom-right corner.

Shown in (skipped during scene transitions):
  - SCENE_GAMETITLE
  - SCENE_GAMEPLAY
  - SCENE_INTRO
  - SCENE_GAMEOVER
  - SCENE_EXPERIMENT  (#if defined(_DEBUG) only)

Layout:
  - Font size: clamp(iOrigHeight / 72, 10, 16)  — 10pt at <=720p, ~15pt at 1080p
  - Colour: MyColor(160, 160, 160, 200)          — dim grey, semi-transparent (v1 only)
  - X: width - estimatedTextWidth - 8px          — right-aligned with 8px margin
  - Y: height - fontSize - 8px                  — 8px above bottom edge
  - X clamped to >= 8px to never overflow left on very small windows

Text width is estimated as textLength x fontSize x 0.6 (proportional font
approximation). Overestimate is intentional so the text stays safely away
from the right edge.

Files changed: DXRenderFrame.cpp, Renderer.h
