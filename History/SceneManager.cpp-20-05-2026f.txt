SceneManager.cpp - 20-05-2026f - Cache-restore correctness overhaul (v0.0.1273)

CHANGES:
ParseGLBScene and ParseGLTFScene cache fast-paths restructured for full correctness.

PROBLEM 1 — wrong document order:
  GLB file was re-read AFTER models were restored from cache. This meant
  m_blenderConfig (axis flip, winding order), exporter detection, camera,
  lights, materials, and animations were all stale when scene_models[] was
  populated. Result: wrong geometry flip, broken animations, missing textures.

  FIX: Restructured both paths to parse the document FIRST. The new order is:
    Step 1 — Open GLB/GLTF and parse JSON + BIN chunk.
             If parse fails, fall through to full parse (logged as LOG_ERROR).
    Step 2 — Restore per-load session state in the same order as the full parse:
             DetectGLTFExporter → BlenderImports::BuildConfig → m_blenderConfig
             ParseGLTFCamera, ParseGLTFLights
             ParseMaterialsFromGLTF, ParseAnimationsFromGLTF
    Step 3 — Restore scene_models[] from models[] cache.
    Step 4 — Rebind textures/materials for all renderers.
    Step 5 — Start animations.

PROBLEM 2 — GPU rebuild and texture rebind were DX11-only:
  The GPU null-check/rebuild block was inside #if defined(__USE_DIRECTX_11__).
  Vulkan and OpenGL had zero GPU validity checks — a Vulkan reload never
  rebuilt missing VkBuffer/VkPipeline and never rebound descriptor sets;
  OpenGL never checked VAO/VBO/EBO handles.

  FIX: Expanded to renderer-specific #if chains:
    DX11/DX12:  check !vertexBuffer || !indexBuffer || !constantBuffer
    Vulkan:     check vertexBuffer/indexBuffer/pipeline == VK_NULL_HANDLE
    OpenGL:     check VAO == 0 || VBO == 0 || EBO == 0
  Write-back to models[] after SetupModelForRendering mirrors each renderer's
  GPU fields (ComPtr for DX11/DX12, raw VkBuffer for Vulkan, GLuint for OpenGL).
  Material/texture clear-and-rebind (BindGLTFMaterialTexturesToModel) and
  write-back to models[] both expanded with matching #if chains.
  Debug log XMStoreFloat4x4/position.z guarded with DX11||DX12 to fix
  compile errors on OpenGL/Vulkan builds.

PROBLEM 3 — silent catch swallowed all parse exceptions:
  catch (...) {} silently dropped JSON parse errors, returning true with
  broken/incomplete scene state.

  FIX: catch (const std::exception&) with LOG_ERROR + fallthrough to full parse.

PROBLEM 4 (GLTF path only) — conditional mini-parse never ran on same-session reloads:
  GLTF path gated the mini-parse on (!bAnimationsLoaded || texturesNeedReload).
  bAnimationsLoaded stays true after first load so the condition was permanently
  false, meaning exporter/material/animation state was never restored on reload.

  FIX: Removed the conditional gate entirely — document is always parsed first
  (same as GLB path now). The texturesNeedReload detection block is also removed.
