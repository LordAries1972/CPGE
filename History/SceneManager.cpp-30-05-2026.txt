// History: SceneManager.cpp — 30-05-2026
// Crash fix: vkCmdBindVertexBuffers with stale freed VkBuffer after SCENE_EXPERIMENT → SCENE_GAMETITLE
//
// ROOT CAUSE (3-part chain):
//   1. CleanUp() calls scene_models[i].DestroyModel() which frees Vulkan GPU buffers and nulls
//      their handles in scene_models[i], but the models[] cache entries (written via CopyFrom
//      during the FIRST ParseGLTFScene) still hold the same now-freed VkBuffer handles
//      (e.g. 0x88f000000088f — valid-looking but freed).
//
//   2. On the NEXT scene load (returning to SCENE_GAMETITLE), the fast-path cache restore
//      calls CopyFrom(models[m]) which copies the stale handles into scene_models[idx].
//      At this point scene_models[idx].vertexBuffer = 0x88f000000088f (freed, not null).
//
//   3. The GPU rebuild check was:
//        if (!bGpuReady && (vertexBuffer==null || indexBuffer==null)) rebuild;
//      Since bGpuReady was true in models[m] and the stale handle appeared non-null,
//      the entire condition was FALSE — SetupModelForRendering was skipped.
//      The render frame then called vkCmdBindVertexBuffers(stale_handle) → driver crash +
//      validation error "Couldn't find VkBuffer Object 0x88f000000088f".
//
// FIX 1 — SceneManager::CleanUp() [line ~89]:
//   After the scene_models[] destroy loop, added a Vulkan-only O(MAX_MODELS) pass that
//   resets bGpuReady = false and nulls ALL GPU handles (vertex, index, uniform, descriptor,
//   pipeline) on every models[] entry that was previously GPU-ready.
//   This guarantees that the next fast-path cache restore copies VK_NULL_HANDLE (not stale
//   freed handles) into scene_models[].
//
// FIX 2 — GPU rebuild check in fast-path cache restore (2 identical locations, both replaced):
//   Was:  if (!bGpuReady && (vertexBuffer==null || indexBuffer==null))
//   Now:  if (vertexBuffer==null || indexBuffer==null || uniformBuffer==null)
//   Removed !bGpuReady dependency — actual handle validity is the definitive check.
//   Added uniformBuffer to the null check (was missing; null uniformBuffer → UBO update
//   silently skipped → wrong transforms / dark models on subsequent scene visits).
//
// FIX 3 — Vulkan write-back in fast-path cache restore (2 identical locations, both replaced):
//   Added uniformBuffer, uniformBufferMemory, uniformBufferMapped to the write-back block
//   (were previously missing). These are created by SetupModelForRendering and must be kept
//   in sync in models[] so that the 4th+ scene visit also has valid UBO handles.
