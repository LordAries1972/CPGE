DX_FXManager.cpp - 08-05-2026
================================
Changes for WarpDotTunnel FX system:

1. Render() update loop: replaced single-type Starfield check with a loop that
   also calls UpdateWarpDotTunnel(fx, deltaTime) for WarpDotTunnel effects.

2. RenderFX() switch: added case FXType::WarpDotTunnel -> calls RenderWarpDotTunnel().

3. StopAllFXForResize(): added block to save/stop WarpDotTunnel alongside Starfield.

4. New function - Init3DWarpDOTTunnel():
   - Creates FXItem of type WarpDotTunnel, populates WarpTunnelData.
   - Staggered ring pre-spawn: ring i starts at fraction (i/density) along tunnel
     so the density is immediately uniform from frame 1.
   - Initial XY computed via FAST_MATH.FastSinCos on pathAngle = pathT * 2PI.

5. New function - StopWarpDotTunnel():
   - erase/remove_if on FXType::WarpDotTunnel && fxID == tunnelID, clears tunnelID.

6. New function - UpdateWarpDotTunnel():
   - Per-ring speed profile: forward = 0.5 + pathT*1.5 (accelerates near camera);
     reverse = 2.0 - pathT*1.5 (decelerates toward focal point).
   - Ring wraps back to origin on reaching destination end.
   - XY centre: ring.cx/cy = startX/Y + kMaxXYRadius * FastSinCos(pathT * 2PI).
   - Spin: spinAngle +/- spinSpeed*dt, wrapped to [0, 2PI].

7. New function - RenderWarpDotTunnel():
   - Builds viewProj = GetViewMatrix() * GetProjectionMatrix().
   - Per dot: 3D world pos -> XMVector3TransformCoord -> NDC cull -> screen coords.
   - Dot size = 1 + pathT*3; colour: dim blue-white far, bright white-blue near.
   - Alpha edge-fade at 8% travel from each end.
   - Drawn via renderer->Blit2DColoredPixel().
