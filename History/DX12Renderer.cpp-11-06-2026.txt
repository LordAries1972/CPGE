DX12Renderer.cpp - History - 11 June 2026
==========================================

Changes made on 11-06-2026:

FIX - CreatePipelineState() TANGENT input layout — E_INVALIDARG (0x80070057) startup crash (pass 1):

  TANGENT was declared as DXGI_FORMAT_R32G32B32_FLOAT (3 floats, 12 bytes) but the vertex
  shader (ModelVertex.hlsl) reads "float4 tangent : TANGENT" (4 components).  D3D12 rejects
  the PSO at creation when the input element provides fewer components than the shader reads.
  Result was HRESULT 0x80070057 / E_INVALIDARG thrown from CreateGraphicsPipelineState,
  aborting engine startup.

  Fix:
    - TANGENT changed to DXGI_FORMAT_R32G32B32A32_FLOAT (float4, 16 bytes, offset 32),
      matching the vertex struct layout: XMFLOAT3 tangent + float tangentW = 16 bytes.
    - BITANGENT input element removed: shader has no BITANGENT input — it computes
      bitangent = cross(N, T) * T.w in the vertex shader body.
    - DX11 ShaderManager.cpp received the same fix on 08-06-2026 when tangentW was added
      to the Vertex struct; DX12Renderer.cpp was missed at that time.

FIX - CreatePipelineState() root signature/sampler mismatch — E_INVALIDARG (0x80070057) startup crash (pass 2):

  ModelPShader.hlsl was updated (adding PCF shadow mapping) but CreateRootSignature() was
  not updated to match.  Three separate issues all existed simultaneously:

  1. Static sampler s2 type mismatch (PRIMARY E_INVALIDARG CAUSE):
     s2 was D3D12_FILTER_ANISOTROPIC (non-comparison sampler) but the pixel shader declares
     "SamplerComparisonState shadowSampler : register(s2)" and calls SampleCmpLevelZero().
     D3D12 validates comparison/non-comparison sampler type at PSO creation and returns
     E_INVALIDARG when they mismatch.
     Fix: s2 changed to D3D12_FILTER_COMPARISON_MIN_MAG_LINEAR_MIP_POINT with
     D3D12_COMPARISON_FUNC_LESS_EQUAL; border colour OPAQUE_WHITE (samples outside the
     shadow frustum return lit = 1.0).

  2. Missing root parameter b6 (ShadowBuffer):
     Shader reads cbuffer ShadowBuffer : register(b6) but root signature had no b6 entry.
     Fix: added root parameter 7 = InitAsConstantBufferView(6, ..., PIXEL) for b6.

  3. SRV descriptor table covered only t0-t5 (6 slots); shader now uses t0-t8 (9 slots):
     Gloss (t6), emissive (t7), and shadow depth (t8) were outside the declared range.
     Fix: descriptor table range expanded from 6 to 9 descriptors; null SRV pre-population
     loop expanded from 6 to 9 slots.

  Additional changes supporting the above:
    - m_shadowBuffer (b6) allocated in CreateConstantBuffers(); useShadowMap initialised
      to 0.0f so shadow rendering is disabled until a shadow map is provided.
    - m_shadowBuffer bound via SetGraphicsRootConstantBufferView in PopulateCommandList.
    - m_shadowBuffer.Reset() added to Cleanup.
    - DX12Renderer.h: DX12_ROOT_PARAM_SHADOW_BUFFER = 7 constant added; m_shadowBuffer
      member added; heap slot comment updated (slots 0-8 null SRVs).

PORT - DrawMyTextCentered IDWriteTextLayout pixel-perfect centering fix ported from DX11.

Previously: Used DWRITE_TEXT_ALIGNMENT_CENTER which relies on DirectWrite's built-in centering
within a layout rectangle — this can produce half-pixel offsets and inconsistent results across
different DPI settings and font sizes.

New approach (matching DX11Renderer.cpp):
  1. Set DWRITE_TEXT_ALIGNMENT_LEADING and DWRITE_PARAGRAPH_ALIGNMENT_NEAR (no auto-centering).
  2. Create IDWriteTextLayout with a large layout box (10000 x 1000).
  3. Call textLayout->GetMetrics(&metrics) to get exact rendered width/height in pixels.
  4. Compute centred position:
       float centredX = position.x + (controlWidth  * 0.5f) - (metrics.width  * 0.5f);
       float centredY = position.y + (controlHeight * 0.5f) - (metrics.height * 0.5f);
  5. Draw using m_d2dContext->DrawText with a tight rect: RectF(centredX, centredY,
       centredX + metrics.width, centredY + metrics.height).

Result: Sub-pixel accurate text centering that matches DX11 output exactly.
