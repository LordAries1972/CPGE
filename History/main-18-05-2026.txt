CHANGE: Startup audio device detection and reporting

Added DetectAndLogAudioDevices() static function (Windows only) which runs immediately
after COM initialisation in WinMain. It enumerates all four standard WASAPI endpoints:

  - Playback  (Default)        [eRender  / eConsole]        — SoundManager, loopback capture
  - Playback  (Communications) [eRender  / eCommunications]  — mic monitor output, TTS
  - Capture   (Default)        [eCapture / eConsole]         — fallback microphone
  - Capture   (Communications) [eCapture / eCommunications]  — headset/gaming mic (preferred)

For each endpoint it reads PKEY_Device_FriendlyName and PKEY_AudioEndpoint_FormFactor from
the device property store and writes both to the debug log. The form factor identifies the
physical hardware type: Headphones, Speakers, Headset, Microphone, HDMI/DisplayPort, etc.

This makes it immediately visible in the log which physical output/input device is active
at the time of launch. Any routing confusion (e.g. audio going to speakers when headphones
are plugged in) will be diagnosable by comparing the logged device names against what
Windows Sound settings show.

Added #include <functiondiscoverykeys_devpkey.h> to the Windows-only include block for
the PKEY_Device_FriendlyName property key.
