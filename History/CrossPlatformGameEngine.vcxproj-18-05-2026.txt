CrossPlatformGameEngine.vcxproj - History - 18-05-2026
=======================================================

NEW: PrintSDKDiagnostics MSBuild target

Added a target that runs BeforeTargets="ClCompile" and prints a
clearly-formatted diagnostic block to the build output showing:

  Vulkan SDK
  - VULKAN_SDK path (from environment variable)
  - vulkan.h present (in $(VULKAN_SDK)\Include\vulkan\)
  - vulkan-1.lib present (in $(VULKAN_SDK)\Lib\)

  Shader Compilers (all from Vulkan SDK Bin\)
  - glslc.exe — GLSL to SPIR-V (Khronos)
  - glslangValidator.exe — GLSL validation + SPIR-V
  - dxc.exe — DirectX Shader Compiler (DXIL/SPIR-V)
  - HLSL d3dcompiler — via Windows SDK (D3DCompile API), no path needed

  OpenGL / GLEW
  - GLEW header directory: .\include\GL\ (glew.h present check)
  - GLEW static lib: .\lib\glew32s.lib (present check)
  - Warning if glew.h absent (Includes.h fallback to bare gl.h noted)

  Warnings
  - If VULKAN_SDK is not set: MSBuild warning
  - If glew.h is missing: MSBuild warning

NEW: __USE_DIRECTX_11__ added to PreprocessorDefinitions (Debug + Release)

CMakeLists.txt already passes __USE_DIRECTX_11__ via target_compile_definitions,
making the CMake build work. The vcxproj had no equivalent, so the
#if defined(__USE_DIRECTX_11__) guard on line 1 of DX_FXManager.cpp evaluated
to FALSE under MSVS 2022, silently skipping the entire file. Added to
<PreprocessorDefinitions> for both configurations so the MSVS build matches
the CMake build. Belt-and-suspenders alongside the source-level fix in
DX_FXManager.cpp.

Files changed: CrossPlatformGameEngine.vcxproj

NEW: Dynamic executable naming via GameName + ActiveRenderer UserMacros

Added <PropertyGroup Label="UserMacros"> with two properties:
  <GameName>CPGE</GameName>          — must match #define GAME_NAME in Includes.h
  <ActiveRenderer>DX11</ActiveRenderer>  — set to DX11 | DX12 | OpenGL | Vulkan

A derived PropertyGroup maps ActiveRenderer to:
  _RendererPrefix  : DX (for DX11/DX12), OpenGL, or Vulkan
  _RendererDefine  : __USE_DIRECTX_11__, __USE_DIRECTX_12__, __USE_OPENGL__, __USE_VULKAN__

Both Debug|x64 and Release|x64 configuration PropertyGroups now set:
  <TargetName>$(_RendererPrefix)$(GameName)</TargetName>

This produces (with defaults):  DXCPGE.exe
Switching <ActiveRenderer> to OpenGL yields: OpenGLCPGE.exe
Switching to Vulkan yields:                  VulkanCPGE.exe

PreprocessorDefinitions in both configurations changed from hardcoded
__USE_DIRECTX_11__ to $(_RendererDefine) so the correct renderer guard
is passed to the compiler automatically when ActiveRenderer changes.

Files changed: CrossPlatformGameEngine.vcxproj, Includes.h, CMakeLists.txt
