Validation Checklist
Build — must pass for all four targets
 All 4 renderer builds compile clean — cmake-build.bat Debug and Release for DX11, DX12, OpenGL, Vulkan. Any LNK2001 unresolved external on MemoryCopy, MemoryZero, or MatrixCopy4x4F means assembly.asm did not assemble or the linker can't see the obj.
 ml64.exe assembles without errors — watch for A2206: missing operator or A2008: syntax error if the MASM item group in the vcxproj wasn't picking up the .asm changes. Check assembly.asm assembles alone if needed.
Higher-risk changes
MemoryCopy(src, dst, len) — parameter order is reversed from memcpy(dst, src, len). Every replacement was made with this in mind, but a typo here silently corrupts GPU data. Quick sanity check: the first argument is always the source struct/buffer, the second is always the mapped GPU pointer or destination array.
MemoryZero uses RDI (non-volatile) and saves/restores it. If a future routine reorders the push/pop it will corrupt the caller's RDI. Visually verify the push/pop are balanced in the assembled listing if you see unexpected register corruption.
MatrixCopy4x4F is a leaf — no push/pop. XMM0–XMM3 are caller-saved so this is correct, but MSVC debug-mode saves them in shadow space; confirm no debug-mode XMM register mismatch under the debugger.
OpenGL platform guard — the #if defined(_WIN64) / #else memcpy / #endif block around the light copy in OpenGLRenderFrame.cpp must not be accidentally removed or the Linux build will lose its light copy entirely.
Vulkan PLATFORM_WINDOWS guard — the view/proj MatrixCopy4x4F calls and the UBO MemoryCopy are both inside #if defined(PLATFORM_WINDOWS). If PLATFORM_WINDOWS is ever undefined on Windows for any reason, those copies silently fall through the #else std::memcpy path — functionally correct but worth knowing.
Runtime checks
 DX11: Run game title screen and gameplay scene — models should render lit and correctly positioned. If the scene is black or models are at wrong positions, the camera CB or light buffer copy went wrong.
 Vulkan: Same scenes — ship should animate correctly. If the ship vanishes or appears at the origin, the VKCameraUBO world/view/proj copy failed.
 OpenGL: Light count shown in debug overlay (F12) should match DX11. If all models appear unlit, the bulk light copy to the UBO is broken.
 All pipelines: FPS should remain at 60. These are hot-path changes so any regression in throughput would show immediately.
